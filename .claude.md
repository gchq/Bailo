# Bailo Project Guidelines

## Language

- Always use British English for spellings and terms (e.g. "colour", "organisation", "authorise", "centre", "licence").

## General Principles

- Prefer clear, readable code over clever or overly compact solutions.
- Write code that is easy for another engineer to understand and maintain.
- Keep functions and classes focused on a single responsibility.

## Formatting & Linting

### Python

- Use **Black** for formatting. Do not manually format code that conflicts with Black.
- Follow **PEP 8** where not overridden by Black.

### JavaScript / TypeScript

- Follow **ESLint** rules configured in the repository.

## Comments & Documentation

- Use docstrings for functions, classes, and methods.
- Keep docstrings concise (1–2 lines) unless additional explanation is genuinely needed.
- Add inline comments only where the code is complex, non-obvious, or could confuse a future reader.
- Do not add comments that simply restate what the code already expresses clearly.

### JavaScript / TypeScript

- Use **JSDoc-style** docstrings.
- Document function purpose and parameters where useful.

Example:

```typescript
/**
 * Returns the user's display name.
 *
 * @param user - The user object containing profile information.
 * @returns The display name shown in the UI.
 */
```

### Python

- Use **reStructuredText (reST)** format for docstrings.

Example:

```python
"""
Return the user's display name.

:param user: User object
:return: Display name string
"""
```
