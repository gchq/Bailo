## Accessibility Statement

This accessibility statement applies to **Bailo** stored at https://github.com/gchq/Bailo. This app is run by the Bailo
team as part of **GCHQ**. We want as many people as possible to be able to use this app. That means we've built the app
so that you can:

- zoom in up to **300%** without the text spilling off the screen
- navigate **all** of the app using just a **keyboard**

We've also made the text in the app (including user guides) as simple as possible to understand.
[AbilityNet](https://mcmw.abilitynet.org.uk/) has advice on making your device easier to use if you have different
needs.

### How accessible this app is

The app is fully compliant with the **Web Content Accessibility Guidelines version 2.2 AA standard**. For detailed
information on compliance please read the full
[**accessibility test report**](/docs/users/accessibility/accessibility-report).

### Reporting accessibility problems with this app

We're always looking to improve the accessibility of Bailo. If you find any problems not listed in this Statement or
think we're not meeting your needs or the needs of others, consider contacting the Bailo team in the contact section
below.

### Enforcement Procedure

The Equality and Human Rights Commission (EHRC) is responsible for enforcing the Public Sector Bodies (Websites and
Mobile Applications) (No. 2) Accessibility Regulations 2018 (the 'accessibility regulations'). Due to the nature of the
UKIC, breaches to the above accessibility regulations should be handled following internal procedures by the
organisation that owns the app. These procedures must be transparent and accessible. **The responsible organisation for
Bailo is GCHQ**. If you are not satisfied with the accessibility of this app, consider contacting the Bailo team using
the contact section below.

### Contacting Us

To report an accessibility issue with Bailo please report it on [Github Issues](https://github.com/gchq/Bailo/issues).

### Technical information about this app's accessibility

Bailo is committed to making its website accessible, in accordance with the Public Sector Bodies (Website and Mobile
Applications) (No. 2) Accessibility Regulations 2018

### Compliance Status

This website is **fully compliant** with the Web Content Accessibility Guidelines version 2.2 AA standard

### How we tested this app

This app was last tested on **02/12/2025**. This test was carried out by the **Bailo** team. We tested all types of
pages within the application. We tested a random selection of content to check that the app displays it in an accessible
way.

You can read the full [**accessibility test report**](/docs/users/accessibility/accessibility-report). This report is
called an 'Accessibility Conformance Report'. It is created by filling in a 'Voluntary Product Accessibility Template Â®'
(VPAT). The VPAT tests if the app complies with WCAG 2.2 criteria.

#### Testing methods used

- Keyboard-only
- Automated testing with **Cypress** (cypress-axe, using axe-core)
- Manual testing (using **Deque axeTools** and **WAVE Browser extensions**)

### Non-accessible content

There are not unresolved compliance issues.

Issues related to accessibility are listed here and triage appropriately. We intend to address the accessibility defects
